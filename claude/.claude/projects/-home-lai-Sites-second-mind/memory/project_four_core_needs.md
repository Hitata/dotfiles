Four core needs identified 2026-04-06. These are not dev tasks — they are the WHY behind everything being built.

1. **Memory** — Notes taken over a lifetime are keywords of significance. The retrieval/vector DB layer exists to give back his own life's signal in compact, searchable form.
2. **Learning** — Multi-threaded, ADHD-style. Needs better input capture AND a system that supports shifting between threads. Building Second Mind IS the learning system.
3. **Singularity / Merge** — Existential. Must be able to delegate to AI at minimum. The higher-self work. If he can't merge with the tech, it's game over.
4. **Mundane + Companion** — Two parts: (a) Claude handles boring labor so he just executes, (b) reminder companion like gym PT Jim Brown — pings at the right time to keep loops alive. Maps to the Reconnection Layer.

**Why:** These are the root motivations. Every feature, surface, and tool should trace back to one of these four.
**How to apply:** When prioritizing work or evaluating new ideas, ask which of the four it serves. If none, it's noise.
